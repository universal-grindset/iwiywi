Where does iwiywi live?

This shapes everything — scraping, AI calls, mobile delivery, and storage all depend on this choice.

A

Fully Local CLI

Runs entirely on your Mac. Scrapes sources on demand, calls Claude API for step classification, stores readings in a local SQLite DB. QR code opens a tiny local web server on your phone (same Wi-Fi). No backend to maintain.

Pros

  • Simple — no server costs
  • Private data
  • Works offline after first fetch

Cons

  • Mac must be on & running for phone QR to work
  • No sync across devices
B

CLI + Hosted Backend

CLI is the interface on your Mac, but a lightweight server (e.g. on Hetzner or Fly.io) runs the daily scrape + AI classification job. Readings served from the cloud — QR works anywhere, not just on Wi-Fi.

Pros

  • QR works on cellular
  • Always-on — no Mac dependency
  • Could share with others in your group

Cons

  • Server to maintain
  • Small ongoing cost
C

Local CLI, Cloud Sync (Hybrid)

CLI runs locally and does the scraping/AI work, but syncs today's readings to a free Cloudflare Worker / GitHub Pages / Gist. QR points to that public URL — phone works anywhere without maintaining a full server.

Pros

  • No server to run 24/7
  • QR works anywhere
  • Near-zero cost

Cons

  • Slightly more setup
  • Content briefly public
